# Тестирование документации приложения "Найди и иди"

## Техническое задание

| ID требования | Описание требования |
|---|---|
| SC21-01 | **Поле ввода названия магазина**<br>- Поле ввода названия магазина принимает любые символы.<br>- Поле содержит плейсхолдер.<br>- Плейсхолдер исчезает при вводе символов.<br>- Минимальное количество символов — 2.<br>- В поле сохраняются результаты предыдущих поисков. |
| SC21-02 | **Поле выбора типа магазина**<br>- Поле выбора типа магазина представляет собой выпадающее меню со следующими вариантами выбора значений:<br>  1. «Продуктовые магазины»;<br>  2. «Магазины стройматериалов»;<br>  3. «Аптеки».<br>- Поле содержит плейсхолдер «Выберите тип магазина».<br>- При наличии выбранного значения оно отображается в поле. |
| SC21-03 | **Кнопка Поиск**<br>- Кнопка "Поиск" активна только в случае заполнения минимум одного поля (некликабельна при отсутствии заполнения обоих полей).<br>- Успешные результаты поиска отображаются в виде флажков на карте.<br>- Карту можно скроллить и зуммировать (стандартные жесты). |
| SC21-04 | **Кнопка Построить маршрут**<br>- Кнопка "Построить маршрут" активна в следующих случаях:<br>  1. Найден результат поиска введенного пользователем названия магазина и выбран его флаг на карте.<br>  2. Выбран тип магазина (маршрут будет построен до ближайшего магазина с указанным типом).<br>- Кнопка содержит иконку в виде изогнутой стрелки, надпись «Построить маршрут» и цвет фона #fff (белый).<br>- При нажатии на кнопку цвет фона меняется на #586d73 (серый) и название кнопки меняется с «Построить маршрут» на «Отменить маршрут». |
| SC21-05 | **Область карты**<br>- Область карты занимает половину области экрана телефона.<br>- Для построения маршрута требуется выполнение условий в требовании SC21-04.<br>- Построенный маршрут можно отменить только нажатием кнопки «Отмена». |
| SC21-06 | **Общие требования к работе приложения**<br>- При повторном открытии приложения отображать незаконченный ранее маршрут.<br>- Отключать точную геопозицию при низком заряде аккумулятора. |

## Анализ нарушений в техническом задании

### SC21-01: Нарушение критерия **Полнота**
**Фраза с ошибкой:** "Поле ввода названия магазина принимает любые символы."

**Пояснение:**
1. Не указано максимальное количество символов, которое можно ввести в поле
2. Не описана обработка специальных символов, эмодзи, пробелов в начале/конце строки
3. Не указано, как система реагирует на ввод недопустимых символов (если они все-таки есть)

### SC21-01: Нарушение критерия **Полнота**
**Фраза с ошибкой:** "В поле сохраняются результаты предыдущих поисков."

**Пояснение:**
1. Не указано, в каком виде сохраняются результаты (выпадающий список, отдельное меню)
2. Не указано количество сохраняемых результатов поиска
3. Не описано, как долго хранятся эти результаты (до закрытия приложения, постоянно)
4. Не указано, можно ли очистить историю поиска

### SC21-03: Нарушение критерия **Однозначность**
**Фраза с ошибкой:** "Кнопка «Поиск» активна только в случае заполнения минимум одного поля"

**Пояснение:**
1. Неясно, что считается "заполнением" для поля ввода названия - достаточно 2 символов (из SC21-01) или любое количество?
2. Неясно, считается ли выбор из выпадающего списка "заполнением" поля выбора типа магазина

### SC21-03: Нарушение критерия **Полнота**
**Фраза с ошибкой:** "Успешные результаты поиска отображаются в виде флажков на карте."

**Пояснение:**
1. Не описано, что происходит при отсутствии результатов поиска
2. Не указано, как отображаются несколько результатов в одном районе
3. Не описано поведение при клике на флажок (должна ли появляться дополнительная информация)
4. Не указано, как отличаются флажки для разных типов магазинов

### SC21-04: Нарушение критерия **Однозначность**
**Фраза с ошибкой:** "Найден результат поиска введенного пользователем названия магазина и выбран его флаг на карте."

**Пояснение:**
1. Неясно, как именно пользователь должен "выбрать флаг" - кликом, тапом?
2. Не описано визуальное выделение выбранного флажка
3. Не указано, можно ли отменить выбор флажка

### SC21-04: Нарушение критерия **Атомарность**
**Фраза с ошибкой:** Весь блок требований SC21-04

**Пояснение:**
1. В одном требовании смешаны описание состояний кнопки ("активна/неактивна"), ее внешний вид и поведение при нажатии
2. Логичнее разделить на отдельные требования: внешний вид кнопки, условия активности, поведение при нажатии

### SC21-04 и SC21-05: Нарушение критерия **Непересекаемость**
**Фраза с ошибкой:** В SC21-04: "название кнопки меняется с «Построить маршрут» на «Отменить маршрут»", а в SC21-05: "Построенный маршрут можно отменить только нажатием кнопки «Отмена»"

**Пояснение:**
1. Противоречие в названии кнопки для отмены маршрута - в одном требовании "Отменить маршрут", в другом "Отмена"
2. Неясно, это одна кнопка с меняющимся названием или разные кнопки

### SC21-05: Нарушение критерия **Полнота**
**Фраза с ошибкой:** "Область карты занимает половину области экрана телефона."

**Пояснение:**
1. Не указано, в какой ориентации экрана (портретной/ландшафтной)
2. Не учтены различные размеры экранов и соотношения сторон
3. Не описано, как располагается область карты относительно остальных элементов интерфейса

### SC21-06: Нарушение критерия **Полнота**
**Фраза с ошибкой:** "При повторном открытии приложения отображать незаконченный ранее маршрут."

**Пояснение:**
1. Не определено, что считается "незаконченным маршрутом"
2. Не указано, как долго хранится информация о незаконченном маршруте
3. Не описано поведение, если при повторном открытии изменилось местоположение пользователя
4. Не указано, можно ли отключить эту функцию
