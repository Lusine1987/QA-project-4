Краткое пояснение по заполненой таблице:

По стадии проведения:
Дымовое (Smoke): Проверка базовой функциональности перед началом глубокого тестирования.
Приемочное (Acceptance): Проверка на соответствия требованиям заказчика/пользователя.
Регрессионное (Regression): Проверка, что новые изменения не сломали существующую функциональность.

По направленности тестового сценария:
Позитивное: Проверка работы системы на корректных данных.
Негативное: Проверка реакции системы на некорректные данные или действия.
Функциональное: Проверка того, ЧТО делает система (функции, сценарии).
Нефункциональное: Проверка того, КАК система это делает (производительность, безопасность, удобство).

По выполнению кода:
Статическое: Тестирование без запуска кода (например, код-ревью, анализ требований).
Динамическое: Тестирование с запуском кода.

По уровню тестирования (новый столбец):
Модульное (Unit): Тестирование отдельных функций или модулей кода.
Интеграционное (Integration): Тестирование взаимодействия между модулями или системами.
Системное (System): Тестирование системы в сборе.

По степени автоматизации (новый столбец):
Ручное (Manual): Тестирование, выполняемое тестировщиком вручную.
Автоматизированное (Automated): Тестирование, выполняемое с помощью специальных скриптов и программ.
