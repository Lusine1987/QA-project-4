# Grey-box тестирование приложения "Найди и иди"

## Чек-лист проверок с доступом к базе данных

### 1. Тестирование данных о магазинах в БД

**Проверки структуры данных:**
- [ ] Проверить наличие и структуру таблицы магазинов
- [ ] Проверить обязательные поля: ID, название, тип, координаты (широта/долгота)
- [ ] Проверить типы данных полей (строковые, числовые, географические)
- [ ] Проверить индексы для оптимизации поиска по названию и типу
- [ ] Проверить ограничения целостности (foreign keys, если есть связи)

**Проверки качества данных:**
- [ ] Проверить отсутствие дубликатов магазинов
- [ ] Проверить корректность координат (в пределах географических границ)
- [ ] Проверить заполненность обязательных полей (нет NULL в ключевых полях)
- [ ] Проверить соответствие типов магазинов заявленным в ТЗ (только 3 типа)

### 2. Тестирование истории поиска (SC21-01)

**Проверки хранения данных:**
- [ ] Проверить таблицу истории поиска в БД
- [ ] Проверить сохранение введенных названий магазинов
- [ ] Проверить метку времени для каждого поискового запроса
- [ ] Проверить ограничение на количество хранимых записей на пользователя
- [ ] Проверить очистку устаревших записей (если предусмотрено)

**Интеграционные проверки:**
- [ ] Сравнить отображаемую историю поиска в UI с данными в БД
- [ ] Проверить, что при выборе из истории поиска выполняется корректный запрос к БД
- [ ] Проверить обработку специальных символов и SQL-инъекций в поисковых запросах

### 3. Тестирование поисковых запросов

**Проверки поиска по названию:**
- [ ] Просмотреть SQL-запросы при поиске по названию магазина
- [ ] Проверить обработку частичного совпадения (LIKE оператор)
- [ ] Проверить чувствительность к регистру в поиске
- [ ] Проверить производительность поисковых запросов (EXPLAIN запросов)

**Проверки поиска по типу:**
- [ ] Проверить запросы при выборе типа магазина
- [ ] Проверить корректность фильтрации по типу магазина
- [ ] Проверить сортировку результатов (по расстоянию, если заявлено)

### 4. Тестирование алгоритма "ближайшего магазина" (SC21-04)

**Проверки логики выбора:**
- [ ] Просмотреть SQL-запрос для поиска ближайшего магазина по типу
- [ ] Проверить расчет расстояния между текущей геолокацией и магазинами
- [ ] Проверить использование географических индексов для ускорения поиска
- [ ] Проверить обработку случаев, когда ближайший магазин не найден

**Валидация данных:**
- [ ] Сравнить расчетное расстояние в БД с фактическим на карте
- [ ] Проверить корректность выбранного "ближайшего" магазина для разных локаций
- [ ] Проверить обработку пограничных случаев (одинаковое расстояние до нескольких магазинов)

### 5. Тестирование маршрутов и состояний (SC21-05, SC21-06)

**Проверки хранения маршрутов:**
- [ ] Проверить таблицу хранения активных/незаконченных маршрутов
- [ ] Проверить структуру данных маршрута (начальная точка, конечная точка, промежуточные точки)
- [ ] Проверить сохранение состояния маршрута при закрытии приложения
- [ ] Проверить восстановление маршрута при повторном открытии

**Проверки целостности данных:**
- [ ] Проверить связь между маршрутами и магазинами в БД
- [ ] Проверить обработку удаления магазина, на который есть активный маршрут
- [ ] Проверить очистку устаревших маршрутов

### 6. Тестирование транзакций и согласованности

**Проверки атомарности операций:**
- [ ] Проверить транзакционность операций поиска и построения маршрута
- [ ] Проверить откат изменений при ошибках в процессе поиска
- [ ] Проверить блокировки при конкурентном доступе к данным магазинов

**Проверки согласованности:**
- [ ] Проверить синхронизацию данных между кэшем приложения и БД
- [ ] Проверить обработку случаев, когда данные в БД изменяются во время активной сессии

### 7. Производительность и оптимизация

**Проверки производительности:**
- [ ] Проанализировать время выполнения поисковых запросов
- [ ] Проверить использование индексов в запросах
- [ ] Проверить нагрузку на БД при множественных одновременных поисках
- [ ] Проверить кэширование часто запрашиваемых данных

**Нагрузочное тестирование БД:**
- [ ] Проверить поведение при большом количестве записей в таблице магазинов
- [ ] Проверить производительность при частых обновлениях истории поиска
- [ ] Проверить стабильность при длительных операциях построения маршрутов

### 8. Безопасность данных

**Проверки защиты данных:**
- [ ] Проверить валидацию входных данных перед записью в БД
- [ ] Проверить защиту от SQL-инъекций в поисковых запросах
- [ ] Проверить разграничение доступа к данным (если есть разные пользователи)
- [ ] Проверить шифрование чувствительных данных (если применимо)

### 9. Интеграционные проверки с картографическим сервисом

**Проверки взаимодействия:**
- [ ] Просмотреть логику передачи координат из БД в картографический сервис
- [ ] Проверить корректность формата географических данных
- [ ] Проверить обработку ошибок при недоступности картографического сервиса
- [ ] Проверить кэширование картографических данных
